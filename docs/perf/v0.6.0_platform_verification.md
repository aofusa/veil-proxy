# v0.6.0 プラットフォーム×arch 検証マトリクス（ビルド / テスト / E2E / perf）

本ドキュメントは v0.6.0 の「全プラットフォーム×arch のビルド・テスト・E2E・perf」検証の
実施結果と、当開発ホストで実行不能な項目の代替検証・環境制約を記録する。

計測ホスト: x86_64 Linux（4 コア・co-tenant 有り・長期稼働）。KVM は同一アーキ（x86_64）
のみ利用可能で、クロスアーキ（aarch64 等）の full-system VM は TCG のみ。

## 1. ビルド（現行 main コード、cargo warning 0）

| プラットフォーム | x86_64 | aarch64 | ビルド経路 / 備考 |
|---|---|---|---|
| Linux glibc（io_uring 既定） | ✅ | ✅ | x86_64=native、aarch64=cargo-zigbuild（**aws-lc-sys が cmake ビルダー経路のため cmake 導入必須**） |
| Linux glibc（epoll reactor） | ✅ | ✅ | `--features epoll`。バックエンド選択は feature/cfg |
| Linux musl | ✅ | ✅ | rust-musl-cross（aarch64 は aws-lc-sys bindgen に `BINDGEN_EXTRA_CLANG_ARGS` sysroot 指定） |
| macOS（darwin, ring provider） | ✅ | ✅ | cargo-zigbuild。universal2 も生成可 |
| Windows（msvc, ring provider） | ✅ | ⚠️ | x86_64=cargo-xwin ✅。aarch64 は cargo-xwin+ring の `-imsvc` 空パス問題（ツールチェーン制約・コード非依存） |
| FreeBSD 14.3（aws_lc_rs provider, kTLS/AIO） | ✅ | — | **VM ネイティブビルド**（`cargo build --release`）。cross は aws-lc-sys/std 制約。aarch64 は当環境にネイティブ手段なし |
| OpenBSD（ring provider, pledge/unveil） | ✅ | — | **VM ネイティブビルド**（Tier3 = prebuilt std 無し、cross-zigbuild 不可）。aarch64 は当環境にネイティブ手段なし |

- x86_64: **6 プラットフォームすべてビルド成功**。
- aarch64: Linux(glibc/musl)・macOS ビルド成功。aarch64-Windows/BSD は**ツールチェーン/ネイティブ環境の制約**（veil コードは当該 arch 対応済み — 同一コードが aarch64-Linux/macOS と x86_64-Windows でビルド成功）。
- 全 feature マトリクス（24 構成）・clippy（default/full）・fmt は x86_64 Linux で warning 0。

## 3. E2E（プラットフォーム別）

- **Linux io_uring / epoll**: 上記 §2 の E2E スイート（バックエンド実起動、HTTP/1.1・HTTP/2・HTTP/3・gRPC・WebSocket・L4 等 533 ケース）で検証。
- **FreeBSD capability mode 静的配信（F-123）**: VM で index 200 / nested 200 / パストラバーサル 403 / panic 0。
- **FreeBSD kTLS（F-126）**: VM で kTLS on/off 双方の静的配信を検証済み。
- **aarch64 full-system io_uring ライブ E2E**: KVM 不可ホストのため TCG が実用不能（Ubuntu/Alpine とも初期化段階で停滞）。→ aarch64-musl クロスビルド成功 + io_uring アーキ非依存（カーネル ABI 共通）で代替（`tools/qemu/README.md`）。

## 4. perf（tools/perf、docs/perf）

- **Linux x86_64 io_uring 非退行**: `docs/perf/README.md` の「v0.6.0 io_uring 非劣化確認」に
  記録。現行 main（F-123 + http3 修正 + `cap_safe_sleep` 反映後）で同一構成
  （`h2_1_ktls_0_lb_kernel_ofc_1`、3 反復、全 Non-2xx=0）を再計測し退行なしを確認:

  | Target | Proto | Req/s (median±stdev) | 同ホスト nginx 比 |
  |---|---|---|---|
  | veil_glibc | HTTP/1.1 | 3180.6 ± 41.5 | 1.45×（nginx 2188.1） |
  | veil_glibc | HTTP/2 | 2848.8 ± 60.1 | 1.19×（nginx 2393.1） |
  | veil_musl | HTTP/1.1 | 3131.9 ± 38.6 | 1.43× |
  | veil_musl | HTTP/2 | 2740.6 ± 67.0 | 1.15× |

  本セッションの変更は `cfg`/FreeBSD 限定で Linux io_uring ホットパス無変更のため、
  値は v0.6.0 公表値と統計的に同一（退行なし）。
- **HTTP/3 / gRPC / WebSocket・kTLS ホスト計測**: `docs/perf/README.md` に既存記録
  （F-129/F-130 の HTTP/3 io_uring 化、HTTP/2 kTLS ホスト h2load 等）。
- **aarch64 / macOS / Windows / BSD の perf**: ネイティブ実行環境が無く当ホストでは計測不能
  （§5）。ビルドは §1 で検証済み。io_uring データプレーンはアーキ非依存。

## 5. 環境制約サマリ（当ホストで実行不能な項目）

- **aarch64 ネイティブ実行**（Linux ライブ E2E / perf、Windows、macOS、BSD）: クロスアーキ KVM 不可。ビルドはクロスコンパイルで検証、実行検証は不可。
- **macOS / Windows ネイティブ実行**: 対応ホストが無くクロスビルド + packaging のみ（ネイティブセキュリティ機構は実機検証不可・保守的プロファイル）。
- **OpenBSD / FreeBSD の aarch64**: 当環境にネイティブ手段なし。

# バックログ: ヘルスチェックの強化 (Active probing, TCP)

## 目的

現状の基本的な HTTP/TLS ヘルスチェックを拡張し、L4 対応および gRPC などの高度なプロトコルベースの異常検知により、より強固な Resilience 基盤を構築する。Envoy に大きく劣る自己修復機能の底上げを図る。

## スコープ案

1. **gRPC Health Checking Protocol 対応**
   - gRPC バックエンドに対する標準的なヘルスチェックのサポート。
2. **Active Probing**
   - 定期的なカスタムプローブの送信と外れ値検知。
3. **TCP ヘルスチェック**
   - L4 プロキシ (F-18) 向けの TCP レベルの監視。

## 受け入れ条件（案）

- gRPC のヘルスチェックプロトコルを用いたバックエンドの死活監視ができること。
- Active Probing のインターバルおよび失敗閾値が設定通りに機能すること。
- TCP バックエンドに対する死活監視が適切に行えること。

## 依存・リスク

- F-06 (resilience/サーキットブレーカー) と役割分担を明確にする（ヘルスチェックは定期・ゆっくり、ブレーカーは速い失敗・一時除外）。
- ヘルスチェックの定期実行がワーカーリソースを過剰に消費しないこと。

# バックログ: キャッシュのPurge機能・制御高度化

## 目的

Proxy Cache の成熟度を向上させ、キャッシュのパージ（無効化）機能や、kTLS/Compression 併用時の制限を緩和することで、nginx クラスの細かい運用制御を実現する。

## スコープ案

1. **Purge API の提供**
   - 管理エンドポイント、または特定の HTTP メソッド/ヘッダを利用したキャッシュパージ機構。
2. **制御設定の高度化**
   - Zone ごとのキャッシュ設定の強化。
   - 無効化戦略・パージの条件（例: ワイルドカード/プレフィックス）のサポート。
3. **制限緩和と統合**
   - Compression と kTLS の併用時のゼロコピー無効化など、アーキテクチャ上の制約を見直し、可能な範囲で最適化を行う。

## 受け入れ条件（案）

- Purge API を呼び出すことで、該当キャッシュが即座に無効化されることをテストで確認する。
- キャッシュヒット率の改善効果や動作に関するテストカバレッジの確保。

## 依存・リスク

- 既存の cache 機能の拡張として実装する。
- キャッシュの非同期無効化処理がワーカーのリクエスト処理レイテンシに悪影響を及ぼさないこと。

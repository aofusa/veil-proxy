# F-47: XDP/eBPF 隔離検証環境の構築とモジュール分離（F-35 残件）

- **優先度**: P3
- **対応状況**: 保留（環境依存）
- **出典**: `docs/artifacts/remaining_tasks_analysis.md` F-35 残件 / アイデア4

## 機能説明・現状

F-35 のユーザースペース最前線防御（accept 段 IP ブロックリスト）は完了済み。NIC ドライバ段での XDP ドロップ（`aya` ベース）は CAP_BPF・対応 NIC・専用カーネル設定を要するため未実装。

## 改修内容

- XDP/eBPF プログラムを別クレート（例: `veil-ebpf`）として分離し、メインの `default = []` / feature 構成を壊さない。
- CI / 開発環境用に `veth` ペア + CAP_BPF 権限付き Docker サンドボックスを整備し、SYN Flood 防御の E2E 検証を可能にする。

## 受け入れ条件

- 通常環境のビルド・テストに影響を与えない（feature / クレート分離）。
- サンドボックス環境で XDP ドロップの E2E が通過。

## 依存・リスク

- CAP_BPF / 対応 NIC / カーネルバージョンという環境制約が強く、汎用開発環境では検証不能。環境が用意できるまで **保留**。

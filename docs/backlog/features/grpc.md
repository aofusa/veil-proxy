# バックログ: gRPC 対応の拡充

## 現状（2026 時点の整理）

- [Cargo.toml](../../../Cargo.toml) に `grpc` / `grpc-web` / `grpc-full` feature が存在し、`src/grpc/` にワイヤ実装がある。
- HTTP/2 前提の gRPC、HTTP/3 との組み合わせは `grpc-full` 等で整理されている。

## ゴール

「プロダクションで gRPC プロキシとして安心して使える」状態に近づける。

## スコープ案

1. **プロトコル互換性**
   - gRPC-Web（ブラウザ経路）のエッジケース（長大メッセージ、trailers、エラーストリーム）。
   - HTTP/2 / H2C / TLS 終端との組み合わせ表を README に明示。

2. **ストリーミング**
   - クライアント / サーバー / 双方向ストリームのタイムアウト、キャンセル、バックプレッシャーとバックエンドプールの整合。

3. **観測性**
   - gRPC ステータス・レイテンシをメトリクス（将来 Prometheus / OTel と連携）に載せる設計。

4. **テスト**
   - 既存 tonic ベースのテストに加え、**不正フレーム・境界長**の回帰テスト。
   - 代表言語のスタブサーバとの相互運用チェックリスト。

## 受け入れ条件（案）

- `grpc` / `grpc-web` 有効ビルドで既存テスト＋追加シナリオが緑。
- README に「対応する gRPC 機能・非対応の明示」セクションがある。

## 依存・リスク

- HTTP/2 実装（`http2` feature）の安定性に依存。
- メッセージサイズ・ヘッダ上限は DoS 面とセットで設計する。

# バックログ: ドキュメントサイト（docs ページ）作成

## 目的

README が長大になる傾向を緩和し、**検索可能・バージョン付き**の公式ドキュメントを提供する。

## スコープ案

1. **ツールチェーン**
   - MkDocs Material、Docusaurus、mdBook 等から選定（Rust プロジェクトでは mdBook が親和性高い）。
   - 英日の配置方針（README との重複をどう避けるか）。

2. **コンテンツ**
   - クイックスタート、設定リファレンス、TLS/kTLS、各 feature フラグ、運用（systemd）、トラブルシュート。
   - `docs/backlog/` は「ロードマップ」としてサイトに載せるか、開発者向けにリポジトリのみ残すかを決定。

3. **公開**
   - GitHub Pages / 自前ドメイン。リリースタグごとのバージョンスナップショット。

## 受け入れ条件（案）

- メイン README から公式ドキュメントへのリンク。
- 少なくとも「インストール」「設定の骨子」「TLS」の 3 セクションがサイト上で完結。

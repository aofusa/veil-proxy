# バックログ: Prometheus メトリクス拡充と feature フラグによる無効化

## 目的

- 観測性を高める一方、**メトリクス収集のコスト**や **攻撃面（エンドポイント公開）** を運用で選べるようにする。

## スコープ案

1. **メトリクス追加**
   - gRPC（ストリーム数、ステータス）、キャッシュ（ヒット率）、ブレーカー（将来）、WASM（実行時間）など、機能ごとに整理した命名規則（プレフィックス `veil_` 等）。

2. **feature フラグ**
   - 例: `metrics` を default off にするか、サブフラグ `metrics-prometheus` でコンパイル時にエクスポータを完全除外（バイナリサイズ・依存削減）。
   - 設定 `config.toml` 側で **ランタイム無効**（エンドポイント非公開・no-op レジストリ）も併用可能に。

3. **ドキュメント**
   - 公開するメトリクス一覧（カーディナリティ注意: 高カーディナリティラベル禁止ポリシー）。

## 受け入れ条件（案）

- メトリクス無効時に **ホットパスに原子操作やハッシュルックアップが残らない**（または分岐予測可能な軽量ガード）。
- README に「推奨ラベル設計」と「無効化手順」。
